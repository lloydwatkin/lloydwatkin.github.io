Publishing

  • The great thing about the internet is that everyone is a publisher and the WWW + HTML has allowed for that
  • But what happens when systems go away?
  • e.g. geocities - a great loss (thankfully archived) of human thought http://archiveteam.org/index.php?title=GeoCities
  • Kyle Drake's Neocities - website about hating carpets - what if this person goes on to invent time travel?
  • Its a large historical archive - what is worthless today may be priceless in future - e.g. Egyptian worker grafitti
  • What happens when a company gets bought up and your site no longer matches their terms?

Social Networking

  • Like many of you, I use a certain social network
  • I put my memories on there talk about, and share, experiences - we share an awful lot about ourselves
  • What happens when that social network goes away? Or decides to trim its database?
  • Treasured memories will be lost - relationships, friends, family, pets, children
  • Sure maybe you can export your data, but you'll never get it in the same form
  • Worse still what if they decided to censor posts on certain topics (e.g. Palestine) or decided to experiment with its users on a psychological level? http://www.theguardian.com/technology/2014/jun/29/facebook-users-emotions-news-feeds

The mega-silos


  • The sad fact is that the "mega silos" can control the internet through their sheer size
  • If EvilCorp decided to build a custom browser, charge for it, stop it working on other browsers unless licences were paid
  • Compare with Firefox needing to support flawed DRM systems or be left behind

>90% of all email goes through the top 10 providers

  • July 2014 technology radar - http://www.thoughtworks.com/radar/#/
  • ...and...?
  • What if provider "A" decided to replace IMAP?
  • Every system involving email, now has to support this new protocol
  • Writing systems becomes more expensive, more error prone
  • Each time the new API is updated new code has to be written, systems have to be updated
  • What happens if they then only allow internal email (compuserve & AOL of the past)
  • Its a mess!

Decentralization is key to a free and open internet

  • Free to innovate
  • Free to share
  • Free of barriers both geographical and financial
  • Two key components DNS and email are decentralized
  • Have been around for 30+ years

Why aren't we decentralized?

Decentralized is 10x harder

Decentralized is less profitable

Decentralized is slower

  • Ulimately you won't be running the software
  • You need to be tolerant of environment, old versions
  • This isn't a problem in a centralized environment
  • You can't/its hard to mine user data for "free" applications
  • You can't show someone adverts on their own systems
  • Retrieving data from a remote systems is slower

XMPP

  • eXtensible Messaging and Presence Protocol (aka Jabber)
  • Started in 1998, at a time of ICQ, AOLIM, Y! Messenger
  • Lots of different accounts
  • Part of the IETF
  • Set of standards documented for all to use

Working with XMPP in nodejs

  • nodejs very appropriate tool for XMPP
  • asynchronus / evented / streams all very useful

node-xmpp-client


            var Client = require('node-xmpp-client')
            var client = new Client({ /* options */ })
            
            client.on('online', console.log.bind)
            client.on('stanza', function(stanza) {
                stanza.attrs.to = stanza.attrs.to
                delete stanza.attrs.from
                client.send(stanza)
            })
  • Create a client connection
  • Can also be used from a browser (BOSH, websocket)
  • Build chat bots, implement custom logic in a client

node-xmpp-component

  • Build components
  • Implement your own business logic
  • Send messages as local users
  • Components exist on a subdomain of the XMPP server
  • Implement your own custom business logic
  • You can send stanzas as local users
  • Transferrable to any XMPP server you decide to use
